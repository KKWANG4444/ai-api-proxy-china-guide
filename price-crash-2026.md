# AI API 成本变化后，怎么重新做技术选型

[![模型目录](https://img.shields.io/badge/模型-以当前控制台为准-blue)](https://www.aifast.club)

模型厂商和聚合平台的计费会持续变化。与其维护一张很快过期的价格表，不如保留一套重新评估的方法。

## 不要只比较 token 单价

总成本还包括：

- 支付和结算条件；
- 网络与部署；
- 多供应商客户端维护；
- 日志、限流和错误处理；
- 故障迁移的人力；
- 图片、音频、视频等非文本调用。

“某平台比官方便宜”只有在采集时间、模型 ID、计费口径和账户地区都明确时才有意义。

## 用真实日志估算

从至少一周的请求日志统计：

```text
input_tokens_per_day
output_tokens_per_day
media_requests_per_day
peak_requests_per_minute
retry_rate
```

再套用付款时控制台的当前规则。不要拿一次演示请求推算整月预算。

## 模型切换成本

统一接口可以减少客户端代码差异，但不能保证不同模型的行为相同。切换前分别测试：

- 工具调用；
- 图片输入；
- streaming；
- JSON schema；
- 上下文长度；
- 错误码和限流行为。

## 核对当前账户与交易规则

账户、价格和交易规则可能调整。预算评估只引用当前控制台与正式服务条款，不保存旧文章中的具体换算。

## 采购前检查

- [ ] 当前模型 ID 已核验；
- [ ] 维护公告已检查；
- [ ] 计费信息来自当前控制台；
- [ ] 已区分余额换算与模型调用费用；
- [ ] 已确认账户地区对应的支付规则；
- [ ] 已核对国际充值的链；
- [ ] 已从真实日志估算预算；
- [ ] 已确认退款、配额、发票和合同条款。
